핵심 요구사항 분석

  • 지능형 통관/물류 관제 — 소비세 방어 로직, 고객 이력 기반 출고 제어, 시나리오별 자동 분리 출고, 강제 출고 예외 처리
  • 정밀 배송 추적 — 일본 현지 물류사 API 연동(사가와, 야마토), 세분화된 배송 상태 시각화, 이메일/LINE 자동 알림
  • 독립 쇼핑몰 솔루션 — 템플릿 기반 원클릭 쇼핑몰 생성, 고유 도메인 연결, 기본 디자인 커스터마이징
  • 멀티몰 아키텍처 — 확장성과 유연성을 갖춘 멀티테넌트 설계로 다수 쇼핑몰 안정 운영
  • 통합 관리자 — 입점사 계정 관리, 공통 정책(통관 기준 금액, 배송비) 설정, 시스템 현황 모니터링

제안 포인트

1. 멀티테넌트 SaaS 아키텍처 전문성

Harmony Link(시니어케어 관리 플랫폼)에서 멀티테넌트 아키텍처를 설계·구현한 경험을 바탕으로, 입점사별 데이터 분리와 공통 정책 관리를 동시에 지원하는 효율적인 아키텍처를 제안합니다. 논리적 테넌트 분리(Schema-per-tenant)로 MVP 단계에서 빠르게 구축하고, 향후 물리적 분리로 확장 가능하도록 설계합니다.

2. 복잡한 비즈니스 로직 구현 역량

Series-B(VC 펀드 관리 플랫폼)에서 VICS 규제 보고 자동화, 복식부기 회계, 다단계 결재 워크플로우 등 복잡한 비즈니스 로직을 성공적으로 구현했습니다. 이 경험을 소비세 방어 로직, 시나리오별 출고 제어, 정산 차감 등 통관/물류 관제 시스템 개발에 직접 활용합니다.

3. 대규모 외부 API 연동 경험

P2P Funding에서 15개 이상의 외부 API(NICE KYC, Seyfert, PayGate 등)를 안정적으로 연동한 경험이 있으며, 일본 물류사(사가와, 야마토) API 통합 연동에 동일한 아키텍처 패턴을 적용합니다.

4. 정부지원사업 맞춤 MVP 전략

K-브랜드 수출플랫폼 육성사업의 예산 구조(정부지원금 60% + 자부담 40%)에 맞추어, 핵심 기능 위주의 MVP를 120일 내 구현하고 Phase 2 고도화 로드맵을 함께 제시합니다. 사업 계획서에 첨부 가능한 상세 견적과 기술 구현 방안을 포함합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다. MVP 범위와 Phase 2 확장 범위를 구분합니다.

1. 프로젝트 배경 및 목적

본 프로젝트는 국내 우수 화장품 브랜드(K-브랜드)의 일본 시장 진출을 지원하기 위해 기획되었습니다. 일본의 복잡한 통관 규정, 특히 합산 과세에 따른 소비세 발생은 국내 기업의 역직구 수출에 큰 장벽입니다.

이에, 지능적인 통관 및 물류 관제 자동화 로직을 탑재하고, 입점사가 손쉽게 독립적인 쇼핑몰을 생성·운영할 수 있는 SaaS형 커머스 플랫폼을 구축합니다. 'K-브랜드 수출플랫폼 육성사업'의 일환으로 MVP 개발을 통해 핵심 기능을 우선 구현하고, 시장 반응에 따라 고도화합니다.

3. 기능 요구사항

3.1 지능형 통관/물류 관제 시스템 (MVP)

  • [필수] 소비세 방어 로직 — 관리자 설정 기준 금액 초과 시 합산 과세 위험 자동 감지 및 출고 제어
  • [필수] 고객 이력 기반 출고 제어 — 일 2~3회 스케줄러(Bull Queue)로 물류사 API 연동, 동일 고객 주문 자동 제어
  • [필수] Scenario A: 항공/해상 복수 배송 채널 자동 분리 출고
  • [Phase 2] Scenario B: 선행 물품 통관 완료 시점 연계 후행 물품 출고
  • [필수] 강제 출고 승인 + 책임소재 명확화 + 소비세 자동 정산 차감
